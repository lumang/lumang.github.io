在树莓派构建自己iot应用，是学习iot技术的一种途径
本文通过一步步说明来指导树莓派初学者，学习如何让你的IoT应用运行起来

项目说明

通过树莓派记录温度数据，当温度达到某个设定值，然后推送通知消息到Android设备或者微信中，所有的工程文件下载请联系作者lumang(lumang0107)获取

这里我们将要学到的内容

  • 设置树莓派
  • 使用温度传感器，DS18B20
  • 使用PIR 传感器
  • 使用温度传感器和PIR传感器
  • 安装一个web服务器存储数据和运行应用
  • 使用 Python PHP MySQL

    必备知识

  • 熟悉linux和终端命令行操作
  • python 基础语法
  • 基础SQL语句

需要以下软件

  • putty
  • win32Disk Image
  • advance IP scanner
  • WAMP server

系统的工作流程

将传感器链接树莓派GPIO，树莓派将传感器数据上传到web server上，然后web服务器将达到设定值标准数据发送到App中或者公众号上

树莓派
↓
温度传感器，PIR传感器
↓
将传感器数据传到web 服务器
↓
通过推送服务将数据推送到手机APP或者微信

传感器介绍

DS18B20
数字温度传感器，因为树莓派智能处理数字信号
左边GND,中间VCC，最右边是输出/数字引脚
通常的使用方法是在VCC 和输出引脚之间需要连接4.7K欧姆电阻
电源引脚需要输入3.3V电压
PIR 传感器
热释电传感器，也是三个引脚，可以有温度的物体接近

关于毕业择业的问题：我认为应该先选择自己职业在选择就业
记得临近大学毕业的时候学校的就业老师说，我们可以先就业再择业，意思就是先把工作落实，在选择自己的职业
首先这样逻辑地有问题，而且问题学校和学生都有
一方面学校为了就业率，让学生先就业再择业，先把就业率提高，自己的任务先完成，不考虑学生的实际需要
另一方面学生自己的问题，没有长远的规划，入学之初，没有考虑自己就业问题，积累个人的择业能力
为什么要择业再就业呢？
就技术行业来说，一个岗位的高级工程师，首先也要至少5-7年的相关工作经验，如果先择业再就业，就会耽误很多时间，转行穷三年

如何避免先就业再择业的情况呢？
首先学校方面入学就要安排就业培训，传授相关就业的知识
然后作为学生自己要积极学习学校的就业课程，通知积极的发掘自己就业方向，不断积累自己职业技能，多参加公司实习，发现自己的兴趣点
没有找到实习的同学
平时可以做一些小项目，巩固知识

为何说这些呢？因为我自己就是先就业再择业，发现其中存在的问题，浪费很多时间在自己不感兴趣的事情上，再次择业也没有相应的积累，一切从头开始，其中的艰辛很多。

所以这里分享下个人对先就业再择业观点的看法

算下来毕业已经五年了，（2013-1018），实际工作4年左右，因为毕业第一年准备考研耽误了一年的工作，想来当时选择复习考研应该是个错误的决定。
因为就个人来说，不是很适合做研究，另外主要的部分还是为了逃避工作才想着考研，可以在学校多待几年再工作。

然后开始工作，第一份工作大概睡前4500，做嵌入式硬件开发工作，主要是为音频切换器开发控制器，这个工作很清闲，没有太多的开发任务
因为产品追求稳定，核心控制部分基本不需要改动，主要工作就是显示UI的修改。然后干了半年左右换了一家公司做嵌入式软件工程师，这次主要

Led跑马灯

每个学习嵌入式开发的同学的第一板子上的程序应该就是跑马灯了
首先初始化led接口，然后在已大循环中，将led进行定时开关操作
通过这个例子可以，了解RT-Thread,线程的创建方式
线程创建语法

定时滴答时钟

系统滴答时钟的作用是方便CPU调度，不同于硬件定时器，不过是以硬件定时器为基础，通过固件库配置好函数。通过rtconfig 来定义系统时钟，从而定义系统滴答时钟

线程状态

这里的线程不同于PC上的线程，这里的线程是其实是任务，这个概念在ucos中也有，有一定基础的比较好理解
rtthread将任务定义为线程，方便理解
rtthread线程一般来说有三种状态
就绪态
挂起态
运行态
就绪态，是万事俱备，只欠东风了，等待CPU空闲就执行
挂起态，是有高优先级的任务来到，CPU将任务挂起，暂时停止运行
运行态，是当前正在运行的线程

计算机视觉系统概述

计算机视觉系统是最近比较热门的研究领域，今天开始给大家介绍下计算机视觉相关的知识。
视觉是人的所有感官中最敏感的一种，人的视觉可以感知环境，而机器的视觉却很难感知环境
为了解决计算机视觉问题，要确定解决问题的方法，
而解决任何问题都有一个通用方法：通过理解每一步的目的来把问题分解成简单和可实现的自步骤

下面通过对计算机视觉系统进行模块划分，来了解解决计算机视觉问题的方法

计算机视觉系统的组成模块划分

数据采集，获取环境信息

通过相机和arduino系统
通常我们用数码相机，当然现在市面上有很多类型相机都可以
比如手机相机，AR相机，VR相机等等
arduino 可以获取其他感官数据，比如声音，压力，距离等

预处理

把准备好的动心进行处理
原始数据有很多噪音
任何情况下，最好过滤下图片
图片数据RGB red green blue ，每个颜色一个字节的数据 
目的为了从数据中获取真实的数据信息

特征提取

计算机视觉优势就是对场景的自动化解释
提取大数据集的小数据

后处理和后置滤波

特征提取后，需要高层次滤波，滤波有各种方式，根据项目需求来选择

识别检测

计算机视觉系统通过解释图像和图像序列来得到结论
检测是一识别的一种基本形式
识别是一个更复杂的术语，也叫分类
识别过程中将物体分成预先分好的几类，或者通过学习分类，这部分设计深度学习的问题
牢记识别和分类的目的

触发动作

计算机视觉系统是个决策系统处理过程，下一步就是执行动作了

连接子模块

解决任何视觉问题的时候，首先要对问题有个全局的考虑，然后把问题分解成一些系列有意义，内部独立的子模块。算法策略上--分而治之
