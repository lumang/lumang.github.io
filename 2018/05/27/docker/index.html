什么是docker

Docker的思想来自于集装箱,利用集装箱在一艘大船上，可以把货物规整的摆放起来。并且各种各样的货物被集装箱标准化了，集装箱和集装箱之间不会互相影响。
优点
启动快
容器内的应用是底层系统的一个进程，相当于在系统中启动一个进程，不占用太多资源
资源使用少
没有不必要的资源
docker 将应用和应用的依赖，打包到一个文件中，运行这个文件，就启动一个docker
有了docker，解决了web应用的环境问题
docker容器可以进行版本管理，复制，等类似于代码管理

在计算机世界中，云计算类似于大船，docker类似于集装箱
前提条件
Ubuntu系统内核版本高于3.10
通过uname -r 查看内核版本
安装最新docker 安装包

wget -q0- https://get.docker.com/ |sh
下载Docker 依赖
安装后成功提示
启动docker 后台服务
sudo service docker start
测试运行hello-world
sudo docker run hello-world
docker ps

镜像加速
/etc/docker/daemon.json
{
“registry-mirror”:[“http://hub-mirrior.c.163.com"]
}

docker run ubuntu:15.10 /bin/echo”hello world”

Ubuntu 16.
是否安装成功
docker version
docker info
把用户加入docker 工作组
sudo usermod -aG docker $USER
启动本机docker 服务
sudo service docker start

列出image 文件
docker image ls
docker image rm 「imagename」
